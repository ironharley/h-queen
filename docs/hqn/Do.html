6.1 Do

Term 'do' - it’s command, defined by verb what server has to do. Each 'do' has own list of 'what's. List of 'do'-es with according 'what's:

  1. set - set session parameters. See 'email'.
  2. account - account administration options. See 'add', See 'rm', See 'chg'.
  3. hub - Server admin feature - hub administration options. See 'start', See 'stop', See 'restart', See 'snooze', See 'kick', See 'status'.
  4. fork - Server admin feature - fork administration options. See 'start', See 'stop', See 'restart', See 'snooze', See 'kick', See 'status'.
  5. service - Server admin feature - service administration options. See 'start', See 'stop', See 'restart', See 'snooze', See 'kick', See 'status'.
  6. bind - Server admin feature - bind administration options. See 'start', See 'stop', See 'restart', See 'snooze', See 'kick', See 'status'.
  7. open - open resource: remote host or local file (for load content as scenary). See 'host', See 'file'.
  8. list - statistical tool, lists of things: vehicles, untransmitted data, emails in account etc.
  9. close - close opened resource(s), still works. No whats.
  10. exit - close opened resource(s), exit client. No whats.
  11. select - very rough analogue of SQL SELECT operator See whats for it.
  12. delete - very rough analogue of SQL DELETE operator See whats for it, see delid. Move data to trash. Recoverable.
  13. undo - Recovers data from trash. NOTE: You MUST remember delete id or put the same operands as according delete.
  14. purge - Remove all early deleted data - unrecoverable ones. No whats.
  15. export - the same select but output to file/URL.
